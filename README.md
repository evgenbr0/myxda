# MYXDA — журнал о мобильной разработке (myxda.ru)

Готовый статический сайт-энциклопедия о мобильной разработке: платформы iOS и Android,
кроссплатформенные технологии, языки и стек, этапы создания приложений, сравнение платформ,
статьи и FAQ. Это информационный портал (журнал), а не продажа услуг. Сделан по техническому
заданию: чистый HTML/CSS/JS, без бэкенда, базы данных и сборщиков. Полностью адаптивный,
работает сразу после копирования на хостинг.

---

## 🚀 Как разместить на хостинге

Скопируйте всю папку `myxda.ru` (файл `index.html` и папку `assets`) в корневую директорию
вашего хостинга — обычно это `public_html`, `www` или `htdocs`. Файл `index.html` должен
лежать в корне, а папка `assets` — рядом с ним. После загрузки сайт сразу откроется по адресу
домена без какой-либо настройки сервера, сборки или установки зависимостей. Для GitHub Pages
рядом уже лежит пустой файл `.nojekyll`. Локально сайт также можно открыть, просто дважды
кликнув по `index.html` в браузере.

  и CSS-мокапом смартфона со светящимся экраном на фоне неоновых блобов.

- **Направления** — нативная разработка, кроссплатформа, backend, UI/UX, тестирование, публикация.
- **Этапы создания приложения** — 6 шагов от идеи до поддержки.
- **Сравнение iOS vs Android** — две колонки по языку, IDE, магазину, аудитории, монетизации.
- **Статьи** — 6 карточек-превью с тегами и временем чтения и фильтром по темам.

заглушками **ХХХХХ** согласно ТЗ. Цены и часы работы как контакты не используются — это журнал.

## 🔧 Что заменить перед запуском

1. Контактные заглушки `ХХХХХ` → реальные e-mail, телефон, Telegram, адрес и реквизиты
   (в `index.html`, включая блок Schema.org в `<head>` и значения `tel:`/`mailto:`).
2. Ссылки «Читать →» в карточках статей — на реальные страницы материалов (сейчас ведут к форме).
3. Форма обратной связи имитирует отправку на стороне браузера. Чтобы письма приходили на почту,
   подключите сторонний обработчик форм (например, Formspree) или свой backend.

## 🛠 Технологии

- Семантический **HTML5**, разметка **Schema.org** (WebSite + Organization) для SEO.
- **CSS3**: кастомные свойства, Grid/Flexbox, glassmorphism (blur + полупрозрачность),
  неоновый градиент, адаптивные сетки, анимации, `prefers-reduced-motion`, `:focus-visible`.
- **Ванильный JavaScript** (без библиотек): фильтр статей по тегам, валидация формы с
  доступностью (aria-invalid, role=alert), мобильное меню с управлением фокусом (focus-trap,
  inert), аккордеон FAQ «открыт только один», анимации по скроллу (IntersectionObserver),
  тост и кнопка «наверх».
- Шрифты **Golos Text** (заголовки), **Inter** (текст) и **JetBrains Mono** (моно-акценты) —
  с поддержкой кириллицы, подключены через Google Fonts (CDN). Есть системные фолбэки —
  сайт остаётся читаемым даже без доступа к CDN.

Внешних зависимостей, требующих сборки или установки, **нет**.
